# Plan 02-02: Canvas Orbit & Web Audio API Integration

<objective>
Desenvolver o visualizador Canvas 2D exibindo órbitas celestes douradas inspiradas em Eternos, animadas a 60fps de acordo com as frequências sonoras lidas de forma bidirecional pelo Web Audio API.
</objective>

<context>
- O microfone fornece o áudio do usuário e o elemento `<audio>` o áudio da IA.
- Precisamos ler dados do espectro de frequência e converter em raio e brilho das runas centrais.
</context>

<tasks>

<task>
  <description>Implementar a inicialização do Web Audio API (app.js)</description>
  <read_first>
    - backend/public/app.js
  </read_first>
  <action>
    Editar `backend/public/app.js`.
    1. Criar uma função `initAudioContext()` disparada no primeiro clique na página ou ao apertar Espaço.
    2. Instanciar `const audioCtx = new (window.AudioContext || window.webkitAudioContext)()`.
    3. Criar os analisadores `userAnalyser` e `iaAnalyser` com `fftSize = 256`.
    4. Conectar o áudio da IA: `audioCtx.createMediaElementSource(playerEl)` conectado ao `iaAnalyser`, e o `iaAnalyser` conectado a `audioCtx.destination` para manter a saída de áudio audível.
    5. Conectar o áudio do Microfone: Quando o microfone estiver ativado no STT, pegar o stream ativo e criar `audioCtx.createMediaStreamSource(stream)` conectado ao `userAnalyser`.
  </action>
  <acceptance_criteria>
    - `backend/public/app.js` contains `createMediaElementSource` e `createMediaStreamSource`
    - O áudio da IA continua tocando (conectado a `destination`)
  </acceptance_criteria>
</task>

<task>
  <description>Construir a animação geométrica 60fps no Canvas (app.js)</description>
  <read_first>
    - backend/public/app.js
  </read_first>
  <action>
    Editar `backend/public/app.js`.
    1. Obter o canvas `#cosmic-canvas` e redimensioná-lo para preencher a janela completa na inicialização e no `resize`.
    2. Implementar a classe `Particle` para gerenciar partículas douradas que saem do centro e somem gradualmente.
    3. No loop `requestAnimationFrame`, recuperar os dados de frequência: `iaAnalyser.getByteFrequencyData(iaData)` e `userAnalyser.getByteFrequencyData(userData)`.
    4. Calcular as médias de volume do usuário e da IA.
    5. Desenhar círculos concêntricos e runas douradas baseadas na média de volume (pulsando tamanho, opacidade e espessura das linhas).
    6. Se o volume estiver alto, gerar novas partículas.
    7. Atualizar a runa central: cor vermelha brilhante se estiver gravando, dourada viva se a IA falar, dourada fosca se pausado.
  </action>
  <acceptance_criteria>
    - `app.js` contains `requestAnimationFrame`
    - O visualizador renderiza círculos concêntricos e atualiza partículas sem crashar
  </acceptance_criteria>
</task>

</tasks>

<must_haves>
  <truths>
    - As órbitas pulsam fisicamente quando o usuário fala ou quando a IA responde.
    - O framerate é mantido estável a 60fps sem vazamento de memória.
  </truths>
</must_haves>
