# Plan 01-02: Edge-TTS Python Helper & API Route

<objective>
Criar o script Python para Edge-TTS e integrá-lo no backend Node via subprocessos.
</objective>

<context>
- O script Python fará a conversão TTS recebendo argumentos do CLI e gerando MP3 no disco.
- O Node chamará este script usando `child_process`.
</context>

<tasks>

<task>
  <description>Criar o helper Python (tts_helper.py)</description>
  <read_first>
    - .planning/phases/01-core-de-voz-e-conexao-ollama/01-CONTEXT.md
  </read_first>
  <action>
    Criar diretório `backend/helpers/`.
    Criar o arquivo `backend/helpers/tts_helper.py`.
    O script deve aceitar 3 argumentos CLI de forma segura (`sys.argv` ou `argparse`): `--text`, `--voice` (padrão `pt-BR-FranciscaNeural`), `--output`.
    Utilizar o módulo `edge_tts` no Python via `asyncio.run(edge_tts.Communicate(text, voice).save(output_file))`.
  </action>
  <acceptance_criteria>
    - `backend/helpers/tts_helper.py` contains `edge_tts.Communicate`
    - O script aceita texto longo (evitar problemas de quote no bash shell)
  </acceptance_criteria>
</task>

<task>
  <description>Criar ttsService.js no Backend</description>
  <read_first>
    - backend/helpers/tts_helper.py
  </read_first>
  <action>
    Criar `backend/services/ttsService.js`.
    Exportar função `generateAudio(text)`.
    1. A função usa `crypto.randomUUID()` para o nome do arquivo (`output_${uuid}.mp3`).
    2. O caminho de destino deve ser `backend/public/assets/tts/`.
    3. Usar `child_process.exec` ou `spawn` para rodar `python3 backend/helpers/tts_helper.py --text "..." --voice pt-BR-FranciscaNeural --output "backend/public/assets/tts/output_${uuid}.mp3"`.
    4. Garantir que o texto passado não quebra comandos shell escapando aspas.
    5. Retornar a URL relativa: `/assets/tts/output_${uuid}.mp3`.
  </action>
  <acceptance_criteria>
    - `backend/services/ttsService.js` contains `child_process`
    - A função retorna uma promise com o path do audio
  </acceptance_criteria>
</task>
