# Plan 01-03: Frontend STT e Orquestração de Eco

<objective>
Implementar a interface rudimentar do STT com Web Speech API, incluindo a lógica para parar a escuta ativamente (muting automático) durante a reprodução do áudio (TTS) retornado pelo backend.
</objective>

<context>
- A lógica do cliente ficará no vanilla JS em `public/app.js`.
- A Web Speech API envia texto em tempo real (interim) e texto final. Quando capturar texto final, o frontend faz o request HTTP para `/api/chat`.
</context>

<tasks>

<task>
  <description>Criar o boilerplate frontend HTML</description>
  <read_first>
    - backend/server.js
  </read_first>
  <action>
    Criar `backend/public/index.html`.
    Incluir um layout básico em tela escura para debug (a UI avançada fica pra Phase 2).
    Adicionar uma div `<div id="status">` para mostrar se o microfone está "Ouvindo" ou "Pausado".
    Adicionar `<audio id="player" controls style="display: none;"></audio>`.
    Incluir a tag de script linkando para `app.js`.
  </action>
  <acceptance_criteria>
    - `backend/public/index.html` contains `<audio id="player"`
  </acceptance_criteria>
</task>

<task>
  <description>Implementar a Web Speech API no Javascript</description>
  <read_first>
    - backend/public/index.html
  </read_first>
  <action>
    Criar `backend/public/app.js`.
    1. Instanciar `const recognition = new (window.SpeechRecognition || window.webkitSpeechRecognition)()`.
    2. Configurar `.lang = 'pt-BR'`, `.continuous = true`, `.interimResults = false`.
    3. No evento `onresult`, capturar o texto do transcrito. Se houver texto válido, parar o microfone ativamente (prevenção antes de enviar API) e fazer um fetch `POST /api/chat` enviando o JSON `{ text: transcrito }`.
    4. Ao receber o retorno, setar o `src` do `<audio id="player">` com a `audioUrl` e chamar `.play()`.
  </action>
  <acceptance_criteria>
    - `backend/public/app.js` contains `webkitSpeechRecognition`
    - `app.js` faz request para `/api/chat`
  </acceptance_criteria>
</task>

<task>
  <description>Gerenciamento de Estado de Eco e Controle de Espaço</description>
  <read_first>
    - backend/public/app.js
  </read_first>
  <action>
    Editar `backend/public/app.js`.
    1. No player de áudio, escutar o evento `onplay`. Quando disparar, garantir que o `recognition.stop()` foi executado para evitar que o microfone ouça o alto-falante.
    2. Escutar o evento `onended` no player de áudio. Quando o áudio acabar, chamar `recognition.start()` automaticamente.
    3. Adicionar um event listener no `document` para `keydown`. Se `event.code === 'Space'`, alternar manualmente o estado do microfone (iniciar ou parar) para o usuário controlar a IA a força.
  </action>
  <acceptance_criteria>
    - `app.js` escuta `player.addEventListener('onended'` ou apenas `ended`
    - `app.js` trata o evento `keydown` da barra de Espaço
  </acceptance_criteria>
</task>

</tasks>
