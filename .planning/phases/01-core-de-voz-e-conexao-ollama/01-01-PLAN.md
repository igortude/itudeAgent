# Plan 01-01: Backend Express Server & Ollama Integration

<objective>
Configurar o servidor Express básico e construir o serviço de conexão com o Ollama, implementando o healthcheck no startup com fallback resiliente para mock e criando o endpoint `/api/chat` para lidar com a geração de texto em PT-BR.
</objective>

<context>
- O Ollama roda localmente na porta 11434. Se offline, o backend deve funcionar normalmente mas usando um sistema local de mock.
- A comunicação usará o modelo configurado localmente (`llama3.1` ou `qwen2.5-coder`).
- O servidor Express rodará na porta 3000 e servirá conteúdo estático na pasta `backend/public/`.
</context>

<task>
  <description>Criar o Serviço do Ollama (ollamaService.js)</description>
  <read_first>
    - .planning/phases/01-core-de-voz-e-conexao-ollama/01-CONTEXT.md
  </read_first>
  <action>
    Criar `backend/services/ollamaService.js`.
    Exportar `checkHealth()` que faz requisição `GET http://127.0.0.1:11434/api/tags` com timeout de 3 segundos. Retornar booleano `true` se ok, `false` se falhar/timeout.
    Exportar `generateResponse(promptText)` que envia `POST http://127.0.0.1:11434/api/generate` passando o prompt. Injetar o prompt do sistema (persona de IA alienígena sutil). Se `checkHealth` falhar antes, retornar uma resposta mockada: "Estou offline no momento. Respondendo em modo de simulação.".
  </action>
  <acceptance_criteria>
    - `backend/services/ollamaService.js` contains `checkHealth()` e `generateResponse(promptText)`
    - Fallback de erro da API retorna string local de simulação
  </acceptance_criteria>
</task>

<task>
  <description>Criar o servidor Express (server.js)</description>
  <read_first>
    - backend/services/ollamaService.js
  </read_first>
  <action>
    Criar `backend/server.js`.
    1. Instanciar o app do Express na porta 3000, usar `cors()` e `express.json()`.
    2. Importar `ollamaService.js`.
    3. Criar `GET /api/health` retornando o status atual do backend.
    4. Criar `POST /api/chat` recebendo `{ text }` no body, passando para `ollamaService.generateResponse(text)` e retornando o texto da resposta.
    5. Servir o diretório `backend/public/` estaticamente usando `express.static('public')`.
  </action>
  <acceptance_criteria>
    - `backend/server.js` contains `app.post('/api/chat',` e `app.use(express.static('public'))`
    - Inicialização usa porta `3000`
  </acceptance_criteria>
</task>
